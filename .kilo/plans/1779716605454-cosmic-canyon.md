## P0: Critical Fixes

### 1. Fix massive images in build output (up to 19 MB)
- **Problem**: Blog hero images processed by Astro's `compile` image service output near-original-size files. `dist/_astro/blog-photo-vis-lidar.TUNFxLux.png` is 19 MB.
- **Fix**: In `astro.config.ts`, configure the image service with format/quality constraints. Update `BlogPostPreview.astro` and any direct `heroImage` usage to enforce WebP output with quality limits. Investigate why PNG images aren't being converted to WebP.
- **Files**: `astro.config.ts`, `src/components/BlogPostPreview.astro`, `src/pages/blog/[slug].astro`

### 2. Fix broken skip link (accessibility)
- **Problem**: `Header.astro:8` skip link targets `#main-content`, but `<main>` in `BaseLayout.astro:62` has no `id="main-content"`.
- **Fix**: Add `id="main-content"` to the `<main>` element in `BaseLayout.astro:62`.
- **Files**: `src/layouts/BaseLayout.astro`

### 3. Fix PhotoSwipe double-loading on gallery pages
- **Problem**: `BaseLayout.astro:86-133` loads PhotoSwipe UMD on every page. `MasonryGallery.astro:164-196` tries to import PhotoSwipe as ES module with `is:inline` (bare specifier won't resolve in browser). Both strategies conflict.
- **Fix**: 
  - Remove PhotoSwipe UMD loading from `BaseLayout.astro` entirely (lines 55-56 CSS, lines 86-133 JS)
  - In `MasonryGallery.astro`, replace the `is:inline` import with a proper `<script>` tag (without `is:inline`) so Vite resolves `import PhotoSwipeLightbox from 'photoswipe/lightbox'` correctly
  - Move PhotoSwipe CSS import into `MasonryGallery.astro` via `<style>` or a component-level `<link>`
- **Files**: `src/layouts/BaseLayout.astro`, `src/components/MasonryGallery.astro`

## P1: High Impact

### 4. Switch to `output: "hybrid"` and prerender all pages
- **Problem**: All 8 non-blog pages route through a 1.3 MB Cloudflare Worker on every request. Only the contact API (`functions/api/submit.ts`) needs server rendering.
- **Fix**: 
  - Change `output: "server"` to `output: "hybrid"` in `astro.config.ts:13`
  - Remove existing `export const prerender = true` from blog pages (now the default under hybrid)
  - The API function in `functions/api/submit.ts` is a Cloudflare Function, not an Astro page — it stays server-rendered automatically
  - Verify all pages still render correctly
- **Files**: `astro.config.ts`, `src/pages/blog/[page].astro`, `src/pages/blog/index.astro`, `src/pages/blog/[slug].astro`, `src/pages/blog/category/[category]/[page].astro`

### 5. Change `client:load` to `client:visible` on below-fold components
- **Problem**: 4 components hydrate immediately despite being below the fold.
- **Fix**:
  - `index.astro:240`: `FeaturedItemsCarousel` → `client:visible`
  - `index.astro:244`: `FaqSectionWithCategories` → `client:visible`
  - `index.astro:252`: `TestimonialSection` → `client:visible`
  - `services.astro:66`: `ServicesGrid` → `client:visible`
- **Files**: `src/pages/index.astro`, `src/pages/services.astro`

### 6. Add `compressHTML: true` to Astro config
- **Fix**: Add `compressHTML: true` to the top-level Astro config.
- **Files**: `astro.config.ts`

### 7. Create `public/_headers` for Cloudflare caching + security
- **Fix**: Create `public/_headers`:
  ```
  /_astro/*
    Cache-Control: public, max-age=31536000, immutable
  /fonts/*
    Cache-Control: public, max-age=31536000, immutable
  /optimized-images/*
    Cache-Control: public, max-age=86400, stale-while-revalidate=604800
  /vendor/*
    Cache-Control: public, max-age=86400, stale-while-revalidate=604800
  /*
    X-Frame-Options: DENY
    X-Content-Type-Options: nosniff
    Referrer-Policy: strict-origin-when-cross-origin
    Permissions-Policy: camera=(), microphone=(), geolocation=()
  ```
- **Files**: `public/_headers` (new file)

### 8. Resolve Tailwind v3/v4 conflict — keep v3, remove v4
- **Fix**: 
  - Remove `@tailwindcss/vite` from `package.json` dependencies
  - Run `pnpm install` to update lockfile
- **Files**: `package.json`

### 9. Optimize og-image.png (783 KB → under 100 KB)
- **Fix**: Convert `public/og-image.png` to a compressed PNG or WebP under 100 KB using sharp or an image tool.
- **Files**: `public/og-image.png`

### 10. Fix brand name in page titles
- **Problem**: `HeadSEO.astro` appends "YQL Drone" — should be "Lethbridge Drone".
- **Fix**: Replace "YQL Drone" with "Lethbridge Drone" in `HeadSEO.astro`.
- **Files**: `src/components/HeadSEO.astro`
